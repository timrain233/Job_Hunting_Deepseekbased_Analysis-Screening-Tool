Job_Hunting_Deepseekbased_Analysis_Screening_Tool
-🌟 核心定位：个人求职实战项目 | 专为运营/产品助理求职打磨
- A lightweight AI job matching and analysis system based on Python & LangChain. Built from scratch in one week, solving real job-hunting pain points, focusing on process optimization, quantitative analysis, business rule design and cost decision-making — core competencies for Operation & Product Assistant roles.
- 本人一周独立从0到1落地本项目，针对跨专业转行求职痛点，摒弃低效人工筛选模式，基于LangChain搭建轻量化、可迭代的AI分析架构。结合个人求职背景定制专属业务规则，完成需求拆解、模型选型、功能开发、问题迭代全流程闭环。
项目核心价值不在于技术开发，而在于职场能力沉淀，完整复刻互联网运营、产品岗的工作思维，所有落地能力均可直接复用在实习工作中。
- 后端框架：Python、LangChain（模块化架构解耦）
- AI能力：DeepSeek大模型、通用OpenAI接口适配
- 辅助能力：网页文本解析、异常容错处理、日志闭环管理
- 针对应届生求职普遍痛点：人工筛选低效、岗位判断主观、通用工具适配性差、模型使用成本失衡，搭建一套低成本、高适配、可迭代、可量化的智能岗位筛选体系。
- 通过精细化模型选型、自定义评分规则、全流程流程优化，实现岗位精准分层匹配，彻底解决盲目求职、主观判岗的问题。
- 模型择优选型：对比多款大模型，最终选用DeepSeek，依托其长文本解析精准、批量处理稳定、高性价比的优势，在满足JD分析、规则匹配业务需求的同时，严控使用成本，实现效果与预算双向平衡。
- 模块化可拓展架构：基于LangChain解耦设计，拆分调度、解析、AI决策核心模块，适配通用OpenAI接口，支持后续快速切换模型、新增业务规则，迭代性极强。
- 定制量化规则体系：结合985/211学历背景、数据分析项目经验、转行运营求职定位，搭建五档分层评分机制，标准化区分核心必投、优选、可投、酌情、规避岗位，告别主观判断。
- 场景化容错优化：针对页面DOM变动、窗口失效、AI打分偏差等各类实操问题，设计多重兜底、重试机制与专属Prompt，大幅提升系统稳定性与匹配精准度。
- 全流程闭环沉淀：实现岗位信息采集、JD解析、AI量化打分、岗位分层筛选、日志记录追溯的完整业务闭环。
-  运营助理核心能力
- 降本增效思维：主动发现机械性冗余工作，通过工具化、流程化方式优化业务链路，实现工作提效。
- 资源分层运营思维：对岗位资源做分级、分类、优先级排序，贴合运营用户分层、内容分层、资源分层的核心工作逻辑。
- 量化复盘能力：将主观经验转化为0-100分标准化数据评判体系，用数据支撑决策、复盘优化。
- 迭代优化意识：持续排查问题、打磨规则、优化流程，具备职场必备的复盘迭代思维。
-  产品助理核心能力
- 需求拆解能力：从真实用户痛点出发，完成业务需求拆解、功能模块设计，贴合产品工作逻辑。
- 权衡决策能力：结合业务场景、落地效果、使用成本，自主完成技术方案与模型选型。
- 业务规则搭建能力：独立设计标准化打分、分级、筛选业务规则，具备基础产品规则设计思维。
- 边界兜底思维：提前预判各类异常边界场景，设计容错、重试、兜底机制，保障业务稳定运行。
- 适配个人土木跨专业转行运营求职场景，精准匹配目标岗位：
- 90-100分｜核心必投：运营助理、电商/数据/策略/增长运营
- 78-88分｜优质优选：用户运营、内容运营
- 70-77分｜常规可投：互联网运营、市场/活动运营、运营专员
- 60-68分｜酌情适配：新媒体运营
- 0分｜精准规避：土木、建筑、工程、施工类岗位
